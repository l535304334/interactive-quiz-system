## 当前设计方向

本项目采用 **"玻璃态 + 蓝紫渐变"** 风格：
- 头部导航：半透明玻璃态 + backdrop-filter
- 主色调：蓝紫渐变（`var(--c-primary)` → `#a78bfa`）
- 卡片：白色表面 + 3 层阴影 + 左侧色条
- 模式横幅：红/蓝/金/紫四色渐变
- 暗色模式自动跟随系统

## 设计质量 Checklist

每个新 UI 组件必须满足至少 4 项：

| # | 要求 | 项目现状 |
|---|------|---------|
| 1 | 清晰的层级对比 | ✅ meta-row / 卡片卡片化 / Hero 区域均通过 |
| 2 | 有意的间距节奏 | ✅ 统一 Token（`--sp-*`） + 非均匀 padding |
| 3 | 深度/层次感 | ✅ 阴影 `--sh-*` + 左侧色条 + backdrop-filter |
| 4 | 排版有字符感 | ⚠️ 全局字体统一但未配对，可改进 |
| 5 | 颜色语义化 | ✅ 红错/绿对/金收藏/蓝复习/紫浏览 |
| 6 | hover/focus/active 态 | ✅ `.nav-mode-btn:hover` / `.btn-on` 等 |
| 7 | grid-breaking 布局 | ⚠️ 首页卡片 + 报告图表有 Bento 潜力 |
| 8 | 动效澄清流程 | ✅ Toast 滑入 / 反馈卡 pop / 模态 scaleIn |
| 9 | 数据可视化一体 | ✅ Chart.js 4 图表 + 配色 Token |

## 禁止模式（本项目特化）

- ❌ 默认灰色卡片网格（已避免：使用 meta-row + 彩色横幅 + 色条）
- ❌ 统一间距的"安全灰白"设计（已避免：多色渐变 + 层次阴影）
- ❌ 模板化 Hero 区域（已避免：自定义 emoji 浮动 + shimmer 动画）
- ❌ 单一装饰色（已避免：5 色模式体系 + 多色图表）

## 新组件开发流程

1. 先定义在现有设计系统中的角色（模式/工具/信息/反馈）
2. 选择语义色（而非随机色值）
3. 复用已有 Token 间距/圆角/阴影
4. 添加 hover 态 + 过渡动效
5. 检查桌面端 + 移动端（`@media (max-width: 600px)` 已覆盖）
