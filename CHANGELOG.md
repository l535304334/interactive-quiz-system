## [v1.0.0] — 2026-07-08

首个正式发布版本。经过 7 轮 RC（Release Candidate）扫描修复，覆盖 XSS 防护、数据完整性、错误处理、可访问性、性能与动画合规等维度。

### 新增功能

- **SM-2 间隔重复算法** — 基于 SuperMemo-2 算法实现智能复习调度；答对拉长间隔（1→6→`round(interval × ef)`），答错次日复习；难度因子 ef 范围 [1.3, 2.5+]，答错降低、答对抬升。复习模式按钮实时显示今日待复习题数。
- **今日学习时长统计** — 在累计时长之外，独立统计当日学习秒数，每日自动归零；计时弹窗大圆环按今日目标完成度动态填充。
- **类别管理** — 支持类别新增 / 重命名 / 删除空类别；删除时同步刷新所有下拉与缓存（`populateSourceFilter` / `populateExamSource` / `refreshCatList` / `renderCategoryManageList`）。
- **全局搜索** — 关键词匹配题干 / 选项 / 答案；输入 `#ID` 可按题号精准定位（如 `#123`）；高亮匹配片段；分批构建索引（每批 500 题，`setTimeout(buildBatch, 0)`）不阻塞主线程。
- **Chart.js 4 图表学习报告** — 30 天答题量折线、正确率趋势折线、各科完成度 + 正确率雷达、错题分布 doughnut；附顽固错题 Top 8 排名（带奖牌）与最强 / 最弱科目 + 智能学习建议。
- **复合模式** — 2 个及以上模式同时开启时进入复合模式，对题目取交集刷题；横幅显示当前组合与"全部退出"按钮。
- **自定义闹钟** — 支持小时 + 分钟设置，可选循环提醒或仅响一次；一键预设 5 分 / 25 分 / 45 分 / 90 分；与"1 小时起每 30 分钟"系统休息提醒以先到者响；弹窗内"再 5 分钟"延迟。
- **自动快照上限提升至 10 份** — 从原先的 5 份提升到 10 份，每份包含错题 / 已做 / 收藏 / 错次 / 题库 / 类别 / SRS 进度 / 学习时长等完整状态。
- **存储配额预警** — 通过 `navigator.storage.estimate()` 检测，超过 10MB 阈值主动 toast 提示导出备份。
- **错题分类筛选** — 错题模式下方提供 全部 / 1 次+ / 2 次+ / 3 次+ / 5 次+ / 10 次+ 六档筛选。
- **键盘模式扩展** — 新增 `V`（复习模式）、`M`（自定义题）、`T`（计时闹钟）、`G`（每日目标）等单键快捷操作。

### 7 轮 RC 修复总结

#### RC1 — XSS 防护审计
- 审计所有 `innerHTML` 赋值点，确保动态用户数据均经 `escapeHtml()` 转义
- 覆盖范围：搜索结果展示、错题 Top 列表、自定义题编辑、快照恢复提示、Toast 文案
- 新增校验：JSON 导入 `try/catch` 包裹 + 格式校验失败显性 toast

#### RC2 — 数据完整性
- 快照瘦身 90%：移除冗余字段，仅保留必要状态
- undo 槽位（`undoSlot`）实现真实回滚，而非简单标记
- 合并导入策略：不清空原数据，按 `id` 去重追加
- 错次计数（`wrongCountMap`）合并逻辑修复：累加而非覆盖

#### RC3 — 错误处理强化
- 所有 `localStorage` 读写操作包 `try/catch`
- 配额超限（`QuotaExceededError`）时自动驱逐最旧快照重试，仍失败则显性 toast
- 文件读取 `reader.onerror` 显性提示
- JSON 解析失败提示具体错误信息（不吞掉异常）

#### RC4 — 可访问性
- 所有模态弹窗补 `role="dialog"` `aria-modal="true"`
- Esc 键关闭全部弹窗（无键盘陷阱）
- 点击弹窗遮罩外区域关闭
- `focusModal()` 自动聚焦弹窗，避免焦点残留
- 全局键盘事件在 input/select/textarea 聚焦时屏蔽（保留 Ctrl+Z 与 Esc）

#### RC5 — 动画与性能合规
- 动画属性白名单：仅 `transform` / `opacity` / `clip-path`
- 移除 `width` / `height` / `top` / `left` / `margin` / `padding` 的动画（进度条 `width` 例外）
- 学习报告关闭时 `chart.destroy()` 释放 Chart.js 实例 + 清空 canvas 容器
- 搜索索引分批构建，避免首屏阻塞

#### RC6 — 数据迁移与兼容
- 启动时 `checkAutoRestore()`：检测到进度为空但存在快照时，弹窗提示恢复
- 兼容旧版 `autoBackupData` 单键存储，自动迁移至多快照索引
- `STORAGE_PREFIX = 'tiku_v8_'` 隔离命名空间，避免与其他应用冲突
- 数据 schema 增加 `version: 'v1.0.0'` 字段，便于后续迁移识别

#### RC7 — 视觉与交互统一
- 设计 Token 体系（`--c-*` / `--r-*` / `--sh-*` / `--fs-*` / `--sp-*`）落地，组件样式禁止硬编码色值
- 4 组胶囊导航布局（辅助 / 模式 / 工具 / 信息）+ 移动端"更多 ▼"折叠
- 玻璃态 + 蓝紫渐变视觉系统，3 层阴影体系
- cubic-bezier 动效统一（`--ease` / `--ease-bounce`）
- `prefers-reduced-motion` 自动降级动画至 0.01ms

### 已知限制

- **单文件体量较大** — 主文件约 2.7MB（含 5944 题内联题库），首次加载需 1–2 秒解析；之后从 localStorage 读取瞬时打开。
- **localStorage 可被清空** — 浏览器清理或隐私模式下数据可能丢失，需用户定期通过"数据 → 导出"备份。
- **无云端同步** — 纯前端设计，不提供账号体系与跨设备同步；跨设备迁移需手动导出 / 导入 JSON。
- **无 CSP / HTTPS 保护** — 本地 `file://` 协议加载，非服务器部署；如需部署到服务器，应自行增加 HTTPS 与 CSP 头。
- **题库内置为临床医学样例** — 内置 5944 题为临床医学分类（体格检查 / 实验室检查 / 问诊诊断 / 共选 / 心电图），其他学科需通过 JSON 导入替换。
- **无服务端题库管理** — 题库增删改全部在前端完成，无后台管理界面。
- **Chart.js 依赖 CDN** — 离线环境下图表功能不可用，但答题核心功能不受影响。

## 版本号规则

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)：

- **主版本号（X.0.0）** — 不兼容的 API / 数据 schema 变更
- **次版本号（1.X.0）** — 向下兼容的功能新增
- **修订号（1.0.X）** — 向下兼容的问题修复

数据导出文件中的 `version` 字段（当前 `v1.0.0`）用于后续版本识别与迁移。
