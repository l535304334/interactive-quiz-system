## 1. 快速开始

本项目是一个**纯前端单文件 HTML 应用**，无需安装、无需联网、无需服务器。

### 启动方式

1. 找到 `[题库]交互式刷题页面.html` 文件
2. **双击**该文件，系统会用默认浏览器打开
3. 页面加载完成后，内置题库会自动初始化到浏览器 `localStorage`
4. 即可开始刷题

> 推荐浏览器：Chrome 80+ / Edge 80+ / Firefox 75+ / Safari 13+

### 界面概览

打开后你会看到：

- **顶部导航**：类别筛选、模式按钮（错题/复习/收藏/浏览/考试）、键盘模式开关、乱序开关、搜索、菜单
- **题目卡片**：题干、选项（A-I）、答题反馈
- **底部状态条**：今日时长、进度、目标
- **导航按钮**：上一题 / 下一题

## 2. 首次使用指引

### 第一步：确认题库已加载

首次打开时，应用会自动加载内置题库（数千道题，已分类）。如果题库为空（例如清空过 localStorage），可从「数据 → 题库管理 → 导入 JSON」恢复。

### 第二步：选择类别

顶部「类别筛选」下拉框中选择想刷的科目（如「体格检查」「共选」「实验室检查」「问诊诊断」「心电图」等），或选择「全部」刷所有科目。

### 第三步：选择模式（可选）

默认是「顺序刷题」模式。如需切换：

- 点「错题」只刷做错的题
- 点「复习」刷到期的 SRS 复习题
- 点「收藏」只刷收藏的题
- 点「浏览」进入速记浏览
- 点「考试」进入模拟考试

模式可**任意组合**（如「错题+复习」取交集），详见下文。

### 第四步：开始答题

- 点击选项（或按键盘 A-I）选择答案
- 系统自动判对错，错题自动加入「错题本」
- 答对后可手动「下一题」或开启自动跳题
- 进度自动保存，关闭浏览器也不会丢失

## 3. 答题模式

### 3.1 五种模式

| 模式 | 按钮 | 图标 | 说明 |
|------|------|------|------|
| 错题模式 | 错题 | 📕 | 只刷做错的题（错题本） |
| 复习模式 | 复习 | 🔁 | 只刷 SRS 算法到期的复习题（顶部按钮有数字角标显示待复习数） |
| 收藏模式 | 收藏 | ⭐ | 只刷收藏的题 |
| 浏览模式 | 浏览 | 📖 | 进入速记浏览界面，支持分页/范围/自定义数量 |
| 考试模式 | 考试 | 📝 | 模拟考试：选类别+题数+时间限制，分页作答，交卷评分 |

### 3.2 复合模式（任意组合）

错题、复习、收藏三种模式可以**同时开启**，系统取**交集**：

- 错题 + 收藏 → 既做错又收藏的题
- 错题 + 复习 → 错题中今天到期复习的题
- 错题 + 复习 + 收藏 → 三重过滤的题

复合模式开启后，顶部会显示**复合横幅**，标注当前激活的模式组合与剩余题数。横幅提供：

- 单独退出某个模式（保留其他）
- 「全部退出 ✕」一键关闭所有模式

### 3.3 错题次数筛选

进入错题模式后，会出现错题次数筛选条：1 次+ / 2 次+ / 3 次+ / 5 次+ / 10 次+，用于精准定位「顽固错题」。

### 3.4 答题辅助功能

- **乱序**：开关后题目顺序打乱（仅当前过滤集合内）
- **键盘模式**：开启后可用 A-I 直接选答案、1-9 作为备选、←→ 切题
- **撤销（Ctrl+Z）**：撤销最后一次答题
- **重做（Backspace）**：重新作答当前题
- **自动跳题**：答对后短暂延迟自动下一题
- **移出错题（X 键）**：当前题移出错题本

## 4. 题库管理

通过「数据 → 题库管理」打开。提供：

### 4.1 导入 JSON

- **覆盖导入**：完全替换当前题库与进度
- **合并导入**：题目按 id 去重合并，进度（错题/已做/收藏/错次/SRS）累加

支持两种 JSON 格式：

1. **完整备份格式**（推荐）：包含 questions、wrong、done、fav、wrongCount、srs、sources、studySec 等所有字段
2. **纯题库数组格式**：`[{id, source, question, options, answer, ...}, ...]`

> 字段校验规则：`question` 必须非空字符串、`options` 必须非空数组、`answer` 必须是 A-I 范围内的字母。不合规的题会被跳过，并在 Toast 中显示跳过数量和原因。

### 4.2 导出 JSON

「导出」会将全部数据（题库 + 进度 + 设置）打包为 `题库备份_YYYY-MM-DD.json` 文件下载到本地。

### 4.3 自定义题

- 「自定义题」入口可添加自己的题目
- JSON 格式与题库一致，可指定 `source`（类别）字段
- 自定义题 id 从 `1000000` 起递增，与内置题不冲突
- 添加后可选择「清空全部自定义题」一键删除

### 4.4 类别管理

「类别管理」提供：

- **新增类别**：输入名称即可（不能与已有重名）
- **重命名类别**：会同步更新该类别下所有题目的 `source` 字段，保持引用完整性
- **删除类别**：仅当该类别下**无题目**时才可删除；若仍有题，需先「删除题目」或转移

### 4.5 删除题目

- 可按类别勾选删除（一次可勾多个类别）
- 删除后会自动清理相关联的错题/已做/收藏/错次记录
- 「清空全部自定义题」可一键清除所有 id≥1000000 的自定义题

### 4.6 重置进度

- 可选择「重置全部科目」或「仅重置某一科目」的进度
- 仅清除已做题记录（doneSet），不影响题库本身
- 操作不可恢复，建议先导出备份

## 5. 搜索功能

### 5.1 打开搜索

- 点击顶部放大镜图标，或按 `/` 键

### 5.2 关键词搜索

输入任意关键词，搜索会匹配**题干 + 选项 + 答案文本**：

- 不区分大小写
- 结果最多显示前 30 条，需细化关键词
- 每条结果显示：类别、题号、题干片段（命中处高亮 `<mark>`）、错次/收藏/已做标签

### 5.3 精准定位 #ID

输入 `#数字`（如 `#1`、`#1234`）按题号精准定位：

- 题号 = 内部 id + 1（即 `#1` 对应 id=0 的题）
- 找到后点击结果直接跳转到该题

### 5.4 跳转

点击任意搜索结果，会自动关闭搜索框并跳转到该题。若当前处于浏览/考试模式会先退出。

## 6. 计时与闹钟

通过「数据 → 计时与闹钟」或按 `T` 键打开。

### 6.1 学习计时

- **今日时长**：当天累计学习秒数，显示在顶部和底部状态条
- **累计时长**：跨天的总学习时长
- 每秒自增，页面隐藏时暂停（避免后台虚增）
- 跨日自动重置今日时长

### 6.2 系统休息提醒

- 学习满 1 小时后，每 30 分钟随机弹一条休息提醒（喝水、活动、深呼吸等）

### 6.3 自定义闹钟

- 可设置小时 + 分钟间隔（最少 1 分钟）
- **循环模式**：开启后按间隔循环提醒，直到手动关闭
- **仅响一次**：关闭循环后，响一次即自动停用
- 闹钟状态实时显示剩余分钟数
- 页面关闭后再打开，过期闹钟会自动顺延

### 6.4 连续打卡天数

- 计时面板显示连续学习天数（`streakDays`）

## 8. 学习报告

通过「数据 → 学习报告」或按 `R` 键打开。报告包含：

### 8.1 总览数据卡

- 已做题、待刷题、错题本、收藏、学习时长

### 8.2 四张 Chart.js 图表

1. **近 30 天答题量**：柱状图，每日 done 数
2. **正确率趋势**：折线图，每日正确率
3. **各科完成度 / 正确率**：分科目对比
4. **错题分布**：分科目错题数量

### 8.3 科目排名

- 🏆 最强科目（完成度最高）
- 📌 最弱科目（完成度最低）

按错误次数排序的前 8 道题，带进度条与奖牌图标（🥇🥈🥉）。

根据当前数据自动生成建议，例如：

- 错误率偏高 → 建议先攻错题本

## 9. SRS 间隔复习

### 9.1 算法

基于 **SM-2 间隔重复算法**（SuperMemo 2）：

- 答错 → 进复习队列，1 天后复习
- 答对 → 间隔逐次拉长（1天 → 6天 → 6×难度系数 → ...）
- 难度系数（EF）初始 2.5，答错降低（最低 1.3），答对升高
- 答错会重置复习进度（rep 归零，间隔回到 1 天）

### 9.2 复习触发

- 每道题记录 `next` 字段（下次复习日期，YYYY-MM-DD）
- 复习模式下，只显示 `next <= 今日` 的题
- 顶部「复习」按钮有数字角标显示今日待复习题数

### 9.3 复习流程

1. 进入复习模式 → 看到今日到期的题
2. 答对 → 间隔拉长，下次复习日期推后
3. 答错 → 间隔重置为 1 天，明天再复习
4. 全部复习完 → 提示「今日无复习任务」

### 9.4 与其他模式组合

复习可与错题/收藏组合，例如「复习 + 错题」= 错题中今天到期的复习题。

## 11. 数据备份与恢复

### 11.1 自动快照

- 每次答题、收藏、错题变更等关键操作后，**自动**触发快照（防抖 2 秒）
- 快照存储在 `localStorage`，**最多保留 10 份**
- 超过 10 份时自动覆盖最旧的
- 存储空间不足时，会自动淘汰最旧快照再重试

### 11.2 手动快照

「数据 → 题库管理 → 立即快照」可手动保存当前进度。

### 11.3 恢复快照

「数据 → 题库管理 → 快照列表」可查看历史快照：

- 每条显示时间戳、错题数、已做题数、收藏数
- 点击「恢复」从该快照恢复（当前未保存数据会被覆盖，需二次确认）
- 点击「删除」移除单个快照（不可撤销）

### 11.4 启动自动恢复

应用启动时检测：

- 若错题/已做/收藏均为空且无今日时长
- 但存在历史快照
- → 弹出恢复提示框，显示备份数据规模，询问是否恢复

### 11.5 导出/导入

详见 [4. 题库管理](#4-题库管理)。建议定期导出 JSON 文件到本地，作为 localStorage 之外的额外备份。

## 12. 常见问题 FAQ

### Q1：我的数据存到哪里了？关机会不会丢？

数据存在浏览器的 `localStorage`，**不上传任何服务器**。只要不主动清除浏览器数据、不清空网站数据，关机重启都不会丢。但浏览器提供商会按源（协议+域名+端口）隔离数据，请始终用**同一方式**打开（如都用 `file://` 双击，或都部署到同一域名）。

### Q2：为什么换浏览器/换电脑后看不到我的数据？

`localStorage` 是浏览器本地存储，不同浏览器、不同电脑之间不互通。迁移数据请用「导出 JSON」→ 在新环境「导入 JSON」。

### Q3：浏览器提示「存储已满」怎么办？

说明 `localStorage` 配额已耗尽（通常 5-10MB 限制）。处理方法：

1. 立即「导出 JSON」备份到本地
2. 在「快照列表」中删除旧的快照释放空间
3. 系统也会自动淘汰最旧快照，但手动清理更彻底
4. 长期建议：定期导出后清空部分历史快照

### Q4：学习报告的图表不显示？

图表依赖 Chart.js v4.4.4，通过 CDN 加载。首次打开报告时需要联网一次拉取并缓存。如离线环境首次使用，请先联网打开一次报告页面。

### Q5：错题本里某道题我已经会了，怎么移除？

三种方式：

1. 在该题上按 `X` 键 → 移出错题本（不影响其他题）
2. 答对该题后，错题会自动按 SRS 算法延期，长期不再出现
3. 「错题模式」下有「清空错题」按钮 → 一键清空全部错题（不可恢复）

### Q6：导入 JSON 报错「文件格式不正确」？

请检查：

1. 文件必须是合法 JSON（可用 `https://jsonlint.com` 校验）
2. 题目数组每项需包含 `question`（非空字符串）、`options`（非空数组）、`answer`（A-I 字母）
3. 不合规的题目会被跳过，Toast 会显示跳过数量和原因
4. 纯题库数组或完整备份格式都可接受

### Q7：自定义题怎么添加？支持哪些字段？

通过「数据 → 自定义题」打开。在文本框粘贴 JSON 数组：

```json
[
  {
    "source": "我的笔记",
    "question": "题目内容",
    "options": ["A.选项1", "B.选项2", "C.选项3", "D.选项4"],
    "answer": "B",
    "qnum": "自定义",
    "subnum": "1"
  }
]
```

- `source` 可省略，会用下拉框选择的类别
- `qnum`/`subnum` 可省略
- 自定义题 id 从 1000000 起，与内置题不冲突

### Q8：考试模式怎么用？

1. 顶部点「考试」或按 `E` 键
2. 选择科目（可选「全部」或「错题本」）
3. 设置题数（如 50/100）
4. 可选时间限制（每题时长 + 总时长）
5. 点开始 → 进入考试界面，分页作答（每页 20 题）
6. 答完点「交卷」→ 自动评分并展示对错

考试模式下不显示答案，交卷后才出结果。

### Q9：暗色模式怎么切换？

应用**自动跟随系统**暗色模式设置。在系统设置中切换深浅色，应用会自动响应，无需手动切换。

### Q10：刷过的进度可以重置吗？会删题库吗？

可以。通过「数据 → 题库管理 → 重置进度」：

- 可选「重置全部」或「仅某科目」
- 仅清除已做题记录（doneSet），**不影响题库本身**
- 错题/收藏/SRS 不受影响
- 操作不可恢复，建议先导出备份

## 13. 注意事项

### 13.1 localStorage 限制

- **同源策略**：`file://` 双击打开与 `http://localhost` 打开被视为不同源，数据不互通
- **配额限制**：通常 5-10MB，超过会触发警告并自动淘汰旧快照
- **可被清除**：浏览器「清除网站数据」会清空所有进度，请定期导出备份
- **隐私模式**：浏览器无痕/隐私模式下 localStorage 可能不持久，关闭即丢

### 13.2 浏览器兼容性

| 浏览器 | 最低版本 | 备注 |
|--------|---------|------|
| Chrome | 80+ | 推荐 |
| Edge | 80+ | 推荐 |
| Firefox | 75+ | 完全兼容 |
| Safari | 13+ | 完全兼容 |
| IE 11 | 不支持 | 缺失 ES6+/Promise/`navigator.storage.estimate` |

### 13.3 文件大小

主文件约 2.7MB（含数千道题与样式脚本），首次加载需 1-2 秒，之后浏览器缓存后秒开。

### 13.4 CDN 依赖

- Chart.js v4.4.4 通过 CDN 加载（仅打开报告时按需拉取）
- 其他所有资源（CSS/JS/题库）均内联，离线可用
- 完全离线场景下首次打开报告会失败，需联网一次缓存

### 13.5 数据安全

- **XSS 防护**：所有动态 HTML 插入均经 `escapeHtml()` 转义
- **不上传**：所有数据本地存储，无任何网络上报
- **导入校验**：JSON 导入有字段完整性校验，恶意数据会被跳过
- **快照保护**：最多 10 份，自动滚动覆盖，存储满时自动淘汰

### 13.6 跨设备使用

如需在多设备同步：

1. 设备 A：导出 JSON 文件
2. 设备 B：导入 JSON 文件（选「合并」或「覆盖」）
3. 注意：合并不去重进度，重复导入会累加错次

### 13.7 移动端支持

- 响应式布局，支持手机/平板
- 题目卡片支持**左右滑动**切题（触摸事件）
- 但移动端无键盘，建议用按钮操作
